Dokumen ini adalah Buku Panduan Resmi dari A sampai Z untuk aplikasi ServicePro HP yang telah dibangun. Dokumen ini mencakup cara penggunaan, daftar fitur, hingga arsitektur teknis.

📱 ServicePro HP v5.1 — Manual Owner

Versi: 5.1 | Platform: Google Apps Script (Web App) | Database: Google Sheets

🏢 GAMBARAN SISTEM

ServicePro HP adalah sistem manajemen toko servis HP multi-cabang berbasis web. Data tersimpan di Google Sheets — tidak perlu install aplikasi, bisa diakses dari HP atau laptop mana saja.

Cabang yang Didukung

Kode Nama Warna
SIB Cabang SIB Biru
BL Cabang BL Ungu
LJ Cabang LJ Emas
AF Cabang AF Merah

Hak Akses (Role)

Role Akses
OWNER Semua fitur + laporan keuangan + void transaksi + semua cabang
ADMIN Semua fitur operasional kecuali laporan laba & void
TEKNISI Input transaksi & update status saja

URL Akses

  • URL /dev → Untuk testing (perubahan langsung aktif)
  • URL /exec → Untuk karyawan (stabil, tidak berubah tiba-tiba)

📋 DAFTAR FITUR LENGKAP

A. OPERASIONAL HARIAN

1. 📊 Dashboard

  • KPI: Total servis hari ini, omzet, stok rendah, transaksi pending
  • Grafik performa per cabang
  • Notifikasi stok hampir habis
  • Filter: Per cabang, per tanggal

2. 💵 Manajemen Kasir

  • Buka laci (input modal awal)
  • Tutup toko (setor kas ke owner)
  • Catat pengeluaran harian (listrik, air, dll)
  • Lihat riwayat kas hari ini (CASH & TRANSFER)

3. ➕ Transaksi Baru (Servis)

  • Input: Nama pelanggan, nomor WA, tipe HP, kerusakan
  • Pilih teknisi dan cabang
  • Tambah part yang digunakan + harga
  • Input ongkos kerja & metode bayar
  • Pilih garansi servis (LCD, Baterai, Kamera, dll)
  • Cetak nota langsung setelah input

4. 🛒 Penjualan Aksesoris

  • Jual aksesoris/part retail (bukan servis)
  • Hitung laba otomatis (harga jual − modal)
  • Terintegrasi ke kas harian

5. 🔄 Update Status

  • Cari transaksi by ID atau nama pelanggan
  • Update status: Antri → Proses → Selesai → Diambil
  • ✏️ Edit Transaksi — ubah nama, HP, kerusakan tanpa hapus
  • 🔧 Edit Part — ubah part/harga tanpa hapus transaksi
  • 🗑️ Hapus/Void (khusus OWNER/ADMIN) — stok dikembalikan, kas di-reverse
  • Lunasi BON (transaksi kredit)
  • Cetak ulang nota

B. STOK & PEMBELIAN

6. 📦 Pembelian Part

  • Input pembelian dari supplier
  • Stok otomatis bertambah
  • Hutang supplier tercatat
  • Terintegrasi kas keluar

7. ↩️ Return Supplier

  • Return part rusak/salah ke supplier
  • Pilih: Cash Refund (kas masuk) atau Kredit
  • Stok otomatis berkurang

8. 🔀 Transfer Cabang

  • Kirim stok antar cabang (misal: SIB → BL)
  • Stok cabang asal berkurang, cabang tujuan bertambah
  • Tercatat di riwayat part

C. DATA & LAPORAN

9. 🗂️ Master Data (Khusus Owner/Admin)

Tab Cabang: CRUD data cabang (nama, alamat, telepon)

Tab Karyawan: CRUD karyawan (nama, peran, cabang, komisi %)

Tab Part/Stok: - Tambah barang baru - Edit nama, harga beli, harga jual, stok, supplier - Hapus/nonaktifkan barang - Filter per cabang

Tab Dropdown: Edit isi dropdown (jenis barang, merk, dll)

Tab Pengguna: Kelola akun login (username, password, role)

10. 💰 Laporan Kas

  • Semua transaksi kas masuk & keluar
  • Filter: per cabang, per tanggal, per metode bayar
  • Input pengeluaran operasional (OPEX)
  • Export CSV

11. 📦 Laporan Stok

  • Daftar semua stok aktif per cabang
  • Indikator 🔴 HABIS (stok = 0) dan kuning (stok ≤ 3)
  • Nilai modal total stok
  • Filter per cabang & tanggal
  • Export CSV → langsung ke Master Data untuk edit

12. 📋 Stock Opname

  • Bandingkan stok sistem vs stok fisik
  • Cetak laporan opname untuk pengecekan manual

13. 📊 Laporan Lengkap

  • KPI: Pendapatan, Pengeluaran Part, Laba Bersih
  • Kinerja per teknisi (jumlah servis, revenue, komisi)
  • Top 10 pelanggan terbaik
  • Daftar transaksi rugi/nihil margin

14. 📈 Laporan Laba Rugi (Khusus Owner)

  • Rincian pendapatan: jasa servis, ongkos kerja, penjualan aksesoris
  • Rincian pengeluaran: modal part, komisi teknisi, OPEX
  • Laba Kotor, Laba Bersih per periode
  • Filter per cabang & tanggal range

15. 🔧 Laporan Servisan

  • Daftar semua unit servis masuk
  • Filter: cabang, status, tanggal, cari nama/ID
  • Breakdown status per cabang (tabel ringkasan)
  • KPI: Total unit, selesai, proses, batal
  • Kolom keuangan: H.Jual Part, Ongkos, Modal, Laba Kotor, Komisi, Laba Bersih
  • Klik baris → langsung buka detail di Update Status

16. 🏠 Ringkasan Tutup Toko (Fitur Utama Owner)

Menjawab: "Hari ini uang di laci harus dibagi berapa?" - 🏭 Uang Part per Supplier → Rp berapa yang harus disisihkan untuk bayar Supplier A, B, dll - 👨‍🔧 Ongkos Jasa per Teknisi → Berapa ongkos yang terkumpul dari masing-masing teknisi - 🛒 Pembelian hari ini → Part yang sudah dibeli & dibayar - 💰 Saldo Kas → Total masuk, keluar, saldo akhir - Panduan visual pembagian: Sisihkan Part + Ongkos Teknisi = Laba Toko

17. 📋 Riwayat Part

  • Semua pergerakan stok: masuk, keluar, transfer
  • Filter: supplier, kategori, merk, tanggal
  • Export CSV

18. 📥 Import Stok

  • Upload CSV untuk input stok massal
  • Mendukung backdate (tanggal pembelian historis)
  • Template CSV tersedia untuk diunduh

19. ⚠️ Pembatalan / Void (Khusus Owner)

  • Void transaksi yang bermasalah
  • Otomatis: stok dikembalikan + kas di-reverse
  • Tercatat di activity log

💰 PANDUAN TUTUP TOKO HARIAN

Alur yang Disarankan:

  1. Buka → Menu Ringkasan Tutup Toko, pilih cabang & tanggal hari ini
  2. Lihat tabel Uang Part per Supplier → pisahkan uang tersebut secara fisik
  3. Lihat tabel Ongkos Jasa per Teknisi → catat untuk perhitungan gaji/komisi
  4. Buka → Menu Manajemen Kasir → klik Tutup Toko (Setor ke Owner)
  5. Input jumlah yang disetor → sistem otomatis hitung sisa laci

Rumus Pembagian:

Uang di Laci = Sisihkan Part + Ongkos Teknisi + Laba Toko + Modal Awal

🔧 TROUBLESHOOTING

Masalah Solusi
Dashboard blank/loading terus Refresh halaman (F5). Jika masih, buka URL /dev
Laporan stok kosong Pastikan data di Master Data → Part/Stok ada & STATUS = AKTIF
Export CSV berantakan Buka di Excel: Data → From Text/CSV → pilih delimiter koma
Stok tidak berkurang setelah transaksi Cek apakah nama part di transaksi sama persis dengan nama di master stok
Transaksi tidak masuk kas Pastikan status diupdate ke SELESAI_DIAMBIL dan metode bayar terisi
Salah input transaksi Update Status → cari ID → tombol ✏️ Edit Transaksi atau 🔧 Edit Part
Transaksi harus dihapus Update Status → cari ID → tombol 🗑️ Hapus/Void (hanya Owner)
Stok salah jumlah Master Data → Tab Part/Stok → Edit → ubah angka stok langsung
Lupa password karyawan Master Data → Tab Pengguna → Edit → ganti password

📱 ServicePro HP v5.1 — Panduan Karyawan

Baca ini sebelum mulai bekerja. Simpan di HP Anda!

🔑 LOGIN

  1. Buka URL yang diberikan oleh Owner
  2. Masukkan Username dan Password Anda
  3. Klik Masuk

⚠️ Jangan berbagi password dengan siapapun!

🏠 HALAMAN UTAMA (DASHBOARD)

Setelah login, Anda akan melihat Dashboard yang menampilkan: - Jumlah servis hari ini - Status transaksi (antri, proses, selesai) - Stok yang hampir habis (perlu dilaporkan ke owner)

📋 PEKERJAAN HARIAN

1. BUKA TOKO (Pagi)

Menu: 💵 Manajemen Kasir 1. Pilih cabang Anda 2. Pilih "Buka Laci (Modal Awal)" 3. Input jumlah uang modal awal yang diterima dari owner 4. Klik Proses Modal Awal

2. INPUT TRANSAKSI SERVIS BARU

Menu: ➕ Transaksi Baru

Langkah: 1. Isi Nama Pelanggan dan Nomor WA 2. Isi Tipe HP (contoh: iPhone 13, Samsung A52) 3. Isi Kerusakan yang dilaporkan pelanggan 4. Pilih Teknisi yang mengerjakan 5. Tambah Part yang digunakan (klik + Tambah Part) - Pilih nama part dari dropdown - Isi harga jual 6. Isi Ongkos Kerja 7. Pilih Metode Bayar (CASH / TRANSFER / BON) 8. Pilih Garansi jika ada 9. Klik 💾 Simpan Transaksi 10. Cetak nota jika pelanggan minta

💡 BON = pelanggan bayar nanti (kredit)

3. UPDATE STATUS SERVIS

Menu: 🔄 Update Status

Gunakan ini untuk: - Memberitahu pelanggan HP sudah selesai - Update ketika pelanggan sudah ambil dan bayar

Langkah: 1. Ketik ID Transaksi atau Nama Pelanggan di kolom cari 2. Klik baris transaksi yang muncul 3. Di panel detail, pilih Status Baru: - ANTRI → baru diterima, belum dikerjakan - SEDANG_PROSES → sedang dikerjakan teknisi - SELESAI_BELUM_DIAMBIL → sudah selesai, belum diambil pelanggan - SELESAI_DIAMBIL → sudah diambil dan sudah dibayar ✅ - BATAL → pelanggan batalkan servis 4. Tambahkan catatan jika perlu 5. Klik 💾 Update Status

Jika ada pelanggan BON yang mau bayar: 1. Cari transaksinya 2. Klik tombol 💰 Lunasi BON 3. Pilih metode pembayaran 4. Klik Konfirmasi

4. JUAL AKSESORIS / PART RETAIL

Menu: 🛒 Penjualan Aksesoris

  1. Pilih nama barang dari dropdown
  2. Isi jumlah yang dibeli
  3. Isi harga jual
  4. Pilih metode bayar
  5. Klik Simpan

5. CATAT PENGELUARAN

Menu: 💵 Manajemen Kasir

Jika ada pengeluaran (beli minum, fotokopi, dll): 1. Pilih cabang 2. Isi Jumlah Pengeluaran 3. Pilih Sumber Dana (CASH atau TRANSFER) 4. Isi Keterangan (wajib! contoh: "Beli air minum") 5. Klik Catat Pengeluaran

6. TUTUP TOKO (Sore/Malam)

Menu: 💵 Manajemen Kasir

  1. Pilih cabang
  2. Pilih "Tutup Toko (Setor ke Owner)"
  3. Isi jumlah uang tunai yang disetor ke owner
  4. Klik Proses Setor Tunai

⚠️ Lakukan ini setiap hari sebelum pulang!

✏️ JIKA ADA KESALAHAN INPUT

Salah nama pelanggan / tipe HP / kerusakan:

  1. Buka 🔄 Update Status
  2. Cari transaksi
  3. Klik tombol ✏️ Edit Transaksi
  4. Ubah data yang salah → Simpan

Salah input part / harga:

  1. Buka 🔄 Update Status
  2. Cari transaksi
  3. Klik tombol 🔧 Edit Part
  4. Ubah part/harga → Simpan

⚠️ Tidak bisa hapus transaksi sendiri — minta bantu Owner!

📦 CEK STOK

Menu: 📦 Laporan Stok

  • Klik 🔍 Tampilkan untuk lihat semua stok
  • Stok merah = HABIS → segera lapor ke owner/admin
  • Stok kuning = hampir habis → perlu restock

❗ HAL YANG DILARANG

Dilarang Alasan
Berbagi akun login Setiap aktivitas tercatat per user
Input transaksi fiktif Semua data diaudit owner
Tidak catat pengeluaran Menyebabkan selisih kas
Tutup toko tanpa klik Tutup di sistem Kas tidak tersinkron

🆘 HUBUNGI OWNER/ADMIN JIKA:

  • Tidak bisa login
  • Ada transaksi yang harus dihapus
  • Stok tidak sesuai dengan fisik
  • Ada error merah di layar
  • Tidak bisa cetak nota

🔄 ALUR DATA UTAMA

Alur Transaksi Servis

Input Form ──► TRANSAKSI (header) + TRANSAKSI_DETAIL (part)
                    │
                    ├──► STOK_PART (STOK berkurang)
                    ├──► RIWAYAT_PART (KELUAR tercatat)
                    ├──► KAS_HARIAN (MASUK saat status=SELESAI_DIAMBIL)
                    └──► PIUTANG (jika metode=BON)

Alur Pembelian Part

Input Pembelian ──► PEMBELIAN (record)
                        │
                        ├──► STOK_PART (STOK bertambah)
                        ├──► RIWAYAT_PART (MASUK tercatat)
                        ├──► KAS_HARIAN (KELUAR jika bayar tunai)
                        └──► HUTANG_SUPPLIER (jika kredit)

Alur Return Supplier

Input Return ──► RETURN_SUPPLIER (record)
                     │
                     ├──► STOK_PART (STOK berkurang)
                     ├──► RIWAYAT_PART (KELUAR tercatat)
                     └──► KAS_HARIAN (MASUK jika cash refund)

Alur Transfer Cabang

Input Transfer ──► TRANSFER_PART (record)
                       │
                       ├──► STOK_PART cabang ASAL (STOK berkurang)
                       ├──► STOK_PART cabang TUJUAN (STOK bertambah)
                       └──► RIWAYAT_PART (TRANSFER_KELUAR + TRANSFER_MASUK)
